## 4. 根本原因分析

### 4.1 胸部センサー単独の限界

**Lyingは100%認識できるのに、Standing/Sittingは失敗する理由**:

1. **Lying（横たわり）の特徴**:
   - 重力方向が明確（Z軸≈±1g、X/Y軸≈0g）
   - 姿勢変化が大きい（立位→横たわりで90度回転）
   - センサー装着位置の影響が小さい

2. **Standing/Sittingの難しさ**:
   - どちらも**上体は直立**（重力方向ほぼ同じ）
   - 違いは**下半身の状態**（膝・腰の角度）
   - 胸部センサーでは下半身の動きを捉えられない
   - 個人差が大きい（姿勢、体型、装着位置）

3. **LOSO validationでの汎化失敗**:
   - 訓練セット（9被験者）のStanding/Sittingパターンが、テスト被験者（1名）に適合しない
   - 被験者固有のパターン（装着角度、体型）が支配的
   - モデルは被験者依存の特徴を過学習

### 4.2 4-Class精度が高く見える理由

**統合による相殺効果**:
- Standing（0）→ Sitting（1）と Sitting（1）→ Standing（0）が相殺
- 4-class評価では両方とも「Stationary（2）」と認識されれば正解
- 結果: **Standing/Sitting個別は0%でも、Stationary統合では100%**

**例（Sub06）**:
- Standing→Sitting: 0% | Sitting→Standing: 100%
- 12-class: Standing=100%, Sitting=0%
- 4-class: Stationary=100%（Standing/Sittingの混同が隠蔽）

### 4.3 ECE超過の原因

**Temperature Scaling の限界**:
- fold7: T=2.784, ECE=0.400（異常に高い）
- Sub08: T=1.380, ECE=0.296
- 原因: 誤分類が多い場合、Temperatureでは補正しきれない
- 根本的な予測精度の低さが反映

**改善策**:
1. **Temperature範囲拡大**: T=0.1-5.0（現在0.5-3.0）
2. **Bin数削減**: 15→10（サンプル不足による不安定性）
3. **Mixup augmentation**: クラス間の境界を滑らかに

## 5. Phase 1移行への影響評価

### 5.1 CCS (Composite Confidence Score) への影響

**CCS計算式**:
```
CCS = 0.6 × U + 0.4 × (1 - S)
```
- U: Uncertainty（正規化エントロピー）
- S: Stability（過去10秒の状態遷移）

**問題**:
1. **U値の不正確性**:
   - Standing/Sittingの混同 → 誤った確信度
   - 例: 実際Standing、予測Sitting（確信度90%）→ U低い → CCS低い → 2000ms選択
   - しかし**誤分類**なので、実際の不確実性は高い

2. **S値の不安定性**:
   - Standing↔Sitting頻繁に切り替わり（モデルの混乱）
   - Stability低下 → CCS上昇 → 誤って500msや100msを選択

3. **BLE interval選択の誤り**:
   - 本来: Stationary（CCS低）→ 2000ms
   - 実際: Stationary→Transition誤認識（CCS中）→ 500ms
   - 結果: **4倍の電力消費**

### 5.2 Phase 1 KPIへの影響

**Phase 1目標**:
- KPI-1: 平均電流 ≥5-10% 削減（vs 100ms baseline）
- KPI-2: Pout(1s) 劣化 ≤+1.0%pt, TL p95 ≤+10%
- KPI-3: 再現性 ±5%, ログロス <1%

**予測される問題**:
- **KPI-1未達**: Sub02/04/08で40%追加消費 → 全体で削減効果相殺
- **KPI-2悪化**: Stationary誤認識 → 不要な頻繁広告 → PDR過度向上だが電力浪費
- **KPI-3困難**: 被験者間変動±14% → 再現性±5%達成不可

### 5.3 移行可否判定

**❌ 現状のままPhase 1移行は不可**:
1. 12-class BAcc未達（0.741 vs 0.80）
2. ECE大幅超過（0.147 vs 0.05）
3. Standing/Sitting誤分類による電力浪費
4. Operational Accuracy 60-78%（Sub02/04/08）

**⚠️ 条件付き移行は検討可能**:
1. **4-class統合運用**:
   - Standing/Sittingを区別せず、「Stationary」として一括処理
   - CCS計算を4-classベースに変更
   - 12-class個別制御は放棄

2. **ECE改善後**:
   - Temperature範囲拡大（T=0.1-5.0）
   - Mixup augmentation実装
   - 目標ECE≤0.08（緩和）

3. **Phase 0-2並行評価**:
   - 腰装着データで同様実験
   - BAcc≥0.90達成ならPhase 0-2を優先

#### Option B: 4-Class統合運用（12-class放棄） ⚠️ **次善策**

**方法**:
- Standing/Sittingを区別せず「Stationary」として統一

**具体策**:

**実装工数**: 1-2日
**期待効果**: ECE 0.147 → 0.08-0.10（目標0.05は困難だが緩和）

   - 12-class個別制御を放棄
   - ECE≤0.10達成（緩和基準）
   - Phase 1実験で4-class統合の有効性を検証

3. **両方失敗の場合**:
   - Phase 1移行を延期
   - Model architecture変更（Transformer, ResNet等）
   - または研究スコープ変更（HAR-drivenからrule-basedへ）

### 7.3 推奨パス

**最優先**: Phase 0-2（腰装着）評価 → 成功ならPhase 1移行

**次善策**: 4-Class統合 + ECE改善 → Phase 1（限定版）

**最終手段**: 研究スコープ変更（HAR精度への依存度低減）

## 8. 論文執筆への示唆

### 8.1 Results セクション

**必須事項**:
1. **Per-class性能の完全報告**:
   - Standing/Sitting個別精度の表を掲載
   - 「4-class精度は高いが、内訳に問題」と明記

2. **混同行列の可視化**:
   - Standing/Sittingの誤分類パターンを図示
   - Stationary→Transition誤認識を強調

3. **BLE制御への実影響を定量化**:
   - Operational Accuracy表を掲載
   - 「最悪ケースで40%誤制御、40%追加電力消費」と明記

### 8.2 Discussion セクション

**重要論点**:
1. **胸部センサー単独の限界**:
   - Lyingは100%認識できるが、Standing/Sittingは困難
   - 下半身の状態（膝・腰角度）を捉えられない
   - 文献比較: 腰装着BAcc>0.90 vs 胸部0.74

2. **LOSO validationの意義**:
   - 被験者間変動を正しく反映
   - 平均性能だけでなく、最悪ケースも報告すべき
   - 「10被験者中6名は良好、4名は失敗」

3. **4-Class統合の妥当性**:
   - Standing/Sitting区別は現実的に困難
   - Phase 1では4-class統合運用を採用した理由

### 8.3 Limitations セクション

**正直に報告すべき制約**:
1. 胸部センサー単独ではpose認識に限界
2. LOSO性能は被験者依存（BAcc range: 0.51-0.91）
3. ECE未達（0.147 vs 0.05）
4. Phase 0-2（腰装着）が本来望ましい

### 8.4 Future Work セクション

**対策の提示**:
1. Multi-sensor fusion（胸部+腰部）
2. Subject-specific adaptation
3. Transformer-based architecture
4. Meta-learning for quick personalization

## 9. 結論

Phase 0-1の包括的分析により、**Standing/Sitting認識の構造的失敗**とそれに起因する**BLE interval制御エラー（最大40%）**が明らかになった。これはPhase 1のエネルギー削減効果を相殺する致命的な問題である。

**推奨される対応**:
1. **即座にPhase 0-2（腰装着）を評価** - 最も確実な解決策
2. **4-Class統合運用を準備** - Phase 0-2失敗時の次善策
3. **ECE改善策を並行実施** - どちらのパスでも必要

**Phase 1移行判断**:
- Phase 0-2成功 → 即座に移行
- Phase 0-2失敗 → 4-Class統合+ECE改善後に限定的移行
- 両方失敗 → 研究スコープ再検討

**論文執筆**:
- Standing/Sitting失敗を正直に報告
- 胸部センサー限界をDiscussionで議論
- Phase 0-2の優位性を示唆
